\chapter{Introduction}

Ce projet porte sur la reproduction et l'analyse critique de l'article
\emph{Dial-In LLM: Human-Aligned LLM-in-the-loop Intent Clustering for
Customer Service Dialogues}~\cite{hong2025dialinllm}. L'objectif general est
double : reconstruire un pipeline experimental fidele au papier lorsque cela
est possible avec les ressources publiques, puis evaluer les limites et les
pistes d'amelioration de cette approche.

\section{Contexte}

L'article etudie un probleme de \emph{intent clustering} dans des dialogues de
service client, avec une approche iterative ou un LLM intervient dans la boucle
pour evaluer la coherence de clusters intermediaires et produire des noms
d'intents. Cette approche est interessante pour un projet de reproduction, car
elle combine plusieurs briques difficiles a realigner correctement :
\begin{itemize}[leftmargin=*]
  \item embeddings de phrases ;
  \item clustering iteratif avec recherche de \texttt{K} ;
  \item echantillonnage representatif des clusters ;
  \item jugement \texttt{Good/Bad} par LLM ;
  \item merge final entre clusters nommes.
\end{itemize}

Le travail deja realise a permis de reconstruire une version executable du
coeur de la methode, de nettoyer le depot pour recentrer le projet sur la
reproduction du papier, et de mettre en place une campagne de benchmark sur les
jeux anglais presentes par les auteurs.

\section{Objectifs du projet}

Le projet s'organise autour de quatre objectifs academiques complementaires :
\begin{enumerate}[leftmargin=*]
  \item s'approprier l'etat de l'art en rapport avec la publication ;
  \item reproduire les resultats fournis par les auteurs de l'article ;
  \item fournir des evaluations complementaires, par exemple avec d'autres
  approches, d'autres jeux de donnees, ou des modifications
  d'hyperparametres ;
  \item proposer des modifications algorithmiques ou computationnelles
  susceptibles d'ameliorer l'approche.
\end{enumerate}

Dans la pratique, les travaux presentes dans ce rapport se concentrent surtout
sur les trois derniers points. L'etat de l'art complet reste un chantier de
redaction a part entiere, mais la reproduction et l'analyse experimentale sont
desormais suffisamment avancees pour soutenir une vraie comparaison entre :
\begin{itemize}[leftmargin=*]
  \item les scores du papier ;
  \item notre reproduction ;
  \item notre version amelioree.
\end{itemize}

\section{Demarche adoptee}

La demarche suivie dans le projet comporte trois etapes principales.

La premiere etape a consiste a fixer une \emph{baseline de reproduction}
solide. Cela a implique l'alignement sur les bons benchmarks, la mise en
place d'une execution stable sur le serveur Jupyter, et l'identification d'une
approximation realiste du protocole des auteurs malgre l'absence de leurs
checkpoints publics.

La deuxieme etape a consiste a mener une \emph{phase d'amelioration ciblee}.
Plutot que de modifier tout le pipeline simultanement, nous avons teste des
leviers separes : \texttt{coherence / naming}, merge final, et selection de
\texttt{K}. Cette decomposition permet d'attribuer plus clairement les gains ou
les regressions a chaque famille de modifications.

La troisieme etape, qui constitue desormais le cadre directeur du projet, est
une \emph{synthese comparative}. L'objectif n'est plus seulement de faire
tourner des variantes, mais de construire une lecture claire de l'etat du
projet : ce qui reproduit le papier, ce qui reste en retrait, et ce que nos
ameliorations apportent reellement.

\section{Structure du rapport}

La suite du document s'organise ainsi :
\begin{itemize}[leftmargin=*]
  \item le chapitre suivant presente la reproduction du papier, le protocole
  retenu, les choix d'implementation et les resultats obtenus ;
  \item le troisieme chapitre decrit les ameliorations proposees, leur logique
  technique et leurs resultats experimentaux ;
  \item le quatrieme chapitre synthetise la comparaison entre papier,
  reproduction et meilleure configuration amelioree ;
  \item le dernier chapitre formule la conclusion generale et les prochaines
  etapes du projet.
\end{itemize}
